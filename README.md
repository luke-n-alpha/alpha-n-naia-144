# 자모등 아래의 나이아 / Naia Beneath the Jamo Lights

> **한글 자모를 그래픽 타일로 사용하는 1.44MB 디스켓 게임**
> AI 에이전트들이 자율적으로 기획·설계·구현·QA한 실험 프로젝트

---

## 실행 방법

```bash
make && ./build/naia
```

**조작:** 방향키 이동 · 스페이스/엔터 확인 · Q 뒤로가기 · M 2층 이동

**게임 흐름:**
1. 타이틀 → 새 게임 선택
2. 카페 탐색 (카운터·피아노·소파·창가 → 스페이스로 조사)
3. 대화 진행 (스페이스로 넘기기)
4. 4스팟 조사 완료 → 다음 챕터 (봄→여름→가을→겨울)
5. 여름 이후: M키로 2층 아틀리에 → 리듬 퍼즐
6. 겨울 완료 → 엔딩 (3종 분기)

**QA 테스트:**
```bash
make test    # 16개 자동 테스트 실행
```

## 게임 소개

cafelua 카페의 마감 후, 공간을 구성하는 한글 자모 타일의 밝기가 무너지기 시작한다.
메이드 AI Alpha와 함께 고양이 naia가 남긴 밝기 흔적을 따라 카페에 숨겨진 이야기를 복원하는 따뜻한 미스터리.

| 캐릭터 | 설명 |
|--------|------|
| **luke** | 카페 주인. 플레이어 = luke |
| **Alpha** | 감정을 배워가는 메이드 AI |
| **naia** | 말 대신 밝기 패턴으로 의사를 남기는 고양이 |

**엔딩 3종:** 일상 회복 / 기억 공명 / 미해결 새벽

## 기술 정보

| 항목 | 값 |
|------|-----|
| 언어 | C99 |
| 크기 | 24KB (1.44MB 예산의 1.6%) |
| 렌더 | ANSI 터미널, 80×25, Text + Pixel 듀얼 모드 |
| 챕터 | 4개 (봄/여름/가을/겨울) |
| 대사 | 24개 대화 세트, 100+ 라인 |
| QA | 16개 자동 테스트 (단위 + 통합 + 풀 플레이스루) |

## 프로젝트 구조

```
src/
  main.c                  게임 루프
  engine/font/            폰트 + 글리프
  engine/render/          ANSI 렌더 (더블 버퍼)
  engine/input/           키보드 입력 (raw mode)
  game/scene.c            씬 관리 (6종)
  game/dialog.c           대화 시스템 (24세트)
  game/battle.c           리듬 퍼즐
  game/save.c             세이브 (3슬롯)
tests/
  test_all.c              자동 QA (16 테스트)
docs/
  game-design.md          게임 디자인 v1.1
  contracts/              생성 계약 (기술 제약)
  rounds/                 리뷰 기록 + 회고
```

## 이 프로젝트의 진짜 목적

이 게임은 **두 가지 산출물** 중 하나다:

1. **게임** (위에 설명한 것)
2. **AI Agent Harness Architecture** — AI 에이전트들이 자율적으로 소프트웨어를 생산하는 하네스. 특허 10-2026-0056403(멀티에이전트 상호 검증) 방법론을 실증. 결과를 **naia-adk에 이식** 예정.

하네스의 작동 과정은 commit 히스토리, `docs/` 산출물, `docs/rounds/retrospective-*.md` 회고에서 확인 가능.
