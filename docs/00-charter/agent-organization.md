## 1. 설계 원칙

1. **좁은 책임**: 각 에이전트는 하나의 도메인만 담당
2. **생성 계약 우선**: 생성 전 공통 스펙(제약)을 먼저 정의
3. **독립성 유지**: 각 에이전트는 다른 에이전트의 존재를 모름 (격리 컨텍스트)
4. **안티앵커링**: 교체 시 이전 에이전트의 기각 이력을 차단
5. **점진 확장**: 3개로 시작, 검증 후 5개로 확장

### 2.3 가상 에이전트 전략

동일 모델을 서로 다른 시스템 프롬프트로 격리하여 "가상 에이전트"로 운용:
- 동일 모델 + 동일 컨텍스트 = 진짜 독립성 약함
- 동일 모델 + 다른 프롬프트/다른 도메인 = 유의미한 독립성
- R=3에서 가상 에이전트로 확장 시 5체제 가능

## 3. 생성 계약 (Creation Contract)

생성 에이전트가 반드시 준수해야 하는 하드 제약. 생성 전에 먼저 정의됨.

### 3.1 사이즈 예산

| 항목 | 할당 | 비고 |
|------|------|------|
| 엔진 바이너리 | ≤ 400 KB | 터미널 + 폰트 렌더러 + 게임 로직 |
| 폰트 데이터 | ≤ 300 KB | 한글 자모 비트맵 (초성+중성+종성) |
| 시나리오/텍스트 | ≤ 200 KB | 압축 후 |
| 사운드 | ≤ 100 KB | 효과음만 |
| 자모 타일셋 | ≤ 100 KB | 밝기 매핑 테이블 |
| 빌드 오버헤드 | ≤ 100 KB | 런타임, 메타데이터 |
| **여유** | **≥ 240 KB** | 1.44MB - 1.2MB |
| **총** | **≤ 1.44 MB** | |

### 3.2 기술 제약

| 제약 | 값 | 비고 |
|------|-----|------|
| 언어 | C 또는 Zig (TBD) | 사이즈 최적화 |
| 플랫폼 | Linux x86_64 | 공모전 요구 확인 필요 |
| 렌더 | 자체 터미널 + 자체 폰트 | 호스트 터미널/폰트 비의존 |
| 텍스트 인코딩 | UTF-8 또는 EUC-KR (TBD) | 한글 자모 조합 |
| 타일 규격 | 8×16 비트맵 | 자모 하나당 |
| 렌더 모드 | 2가지 (글자 모드 / 픽셀 모드) | 듀얼 렌더 |
| 프레임레이트 | ≥ 30 fps (목표 60) | |

### 3.3 인터페이스 계약

모듈 간 인터페이스는 생성 전에 오케스트레이터가 정의. 각 생성 에이전트는 이 인터페이스만遵守:

```
[터미널 엔진] ← framebuffer API → [폰트 렌더러]
[폰트 렌더러] ← 자모 비트맵 요청 → [타일셋]
[게임 로직] ← 화면 출력/입력 → [터미널 엔진]
[게임 로직] ← 시나리오 데이터 → [시나리오 모듈]
```

(인터페이스 상세는 생성 단계에서 구체화)

## 4. 역할 매트릭스

### 4.1 생성 단계

| 산출물 | 생성 담당 | 검증 담당 | 비고 |
|--------|----------|----------|------|
| 엔진 아키텍처 설계 | CX + GE (독립 생성) | OC (merge arbiter) | 보팅으로 최적안 선택 |
| 터미널 엔진 코드 | CX | GE + OC | |
| 폰트 렌더러 코드 | CX | GE (시각 검증) | |
| 한글 자모 타일셋 | GE | CX (사이즈 검증) | |
| 게임 로직 | CX + GE (독립 생성) | OC | |
| 시나리오/텍스트 | GE | OC | 한국어 품질 |
| 빌드/사이즈 최적화 | CX | OC | 1.44MB 예산 |

### 4.2 검증 단계 (특허 메커니즘 적용)

| 검증 유형 | 검증자 | 메커니즘 |
|-----------|--------|----------|
| 코드 정확성 | CX + GE | 독립 리뷰 → 투표 |
| 사이즈 예산 준수 | CX | 빌드 측정 → 임계값 검사 |
| 시각 회귀 | GE | 자모 렌더링 출력 비교 |
| 아키텍처 일관성 | OC | 인터페이스 계약 준수 확인 |
