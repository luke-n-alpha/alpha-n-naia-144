## 1. 비전 (Vision)

> **"한글 자모로 그려진, 자기 자신을 그리는 터미널 게임. 1.44MB 디스켓 한 장에."**

- 한글의 기하학적 미학을 **게임 그래픽 시스템 자체로 사용**하는 시도.
- 터미널·폰트·렌더 엔진까지 자급자족하여 **호스트 환경 의존 0**, 디스켓 한 장으로 완결되는 작품.
- AI(멀티 도구 앙상블)로 어떻게 게임을 개발하는가의 **살아있는 공개 사례**.

- **자체 터미널 엔진** (libtsm fork 또는 직접 구현)

## 5. 차별점 (Differentiators)

선례인 `doom-ascii` 등 텍스트 둠 프로젝트와의 차이:

| 항목 | 기존 텍스트 둠 | 본 프로젝트 |
|---|---|---|
| 터미널 | 호스트 의존 | **자체 터미널 엔진** |
| 폰트 | 호스트 폰트 | **내장 비트맵 폰트** |
| 문자 | ASCII brightness ramp | **한글 자모 = glyph 타일셋** (shape matching) |
| 렌더 | 단방향 픽셀→글자 | **글자 ↔ 픽셀 듀얼 모드** |
| 재사용성 | 둠 전용 | **범용 게임 엔진** |
| 사이즈 | 5MB+ | **1.44MB** |
| 콘텐츠 | id Software 둠 | **오리지널** (cafelua/naia) |

## 6. AI 통합 정책

- **본체는 오프라인 자급자족** (C1 정신). API 호출 의존 금지.
- **룰베이스 NPC**: markov chain, GOAP, dialogue tree 등 ≤ 300KB로 게임 내 AI 구현.
- **옵션 모드** (`--ai` 플래그 등): 사용자가 API 키 제공 시에만 외부 LLM 호출. 디폴트 OFF, 게이트 안내 명시.
- **개발 과정의 AI 사용은 별도**: Claude/Codex/Gemini/GLM/Opencode 등을 개발 도구로 적극 사용. 이는 G2의 핵심이며 commit·문서로 공개.
