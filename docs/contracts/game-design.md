# Game Design Creation Contract — alpha-n-naia-144

**상태:** v1.0 (OC 작성)
**목적:** CX, GE에게 동일 계약으로 독립 게임 디자인 생성 지시

---

## 제약 (반드시 준수)

1. 플랫폼: Linux 터미널, ANSI escape sequence
2. 해상도: 80x25 셀 그리드
3. 언어: C99
4. 총 크기: ≤ 1.44MB (바이너리 + 데이터)
5. 렌더: Text mode (대화/탐색) + Pixel mode (전투/이펙트) 듀얼
6. 폰트: 자체 8x16 비트맵, 한글 자모 68개 + ASCII 95개 + 박스드로잉 40개
7. 색상: 16색 (4-bit)
8. 오디오: v1 무음
9. 입력: 키보드 (방향키 + 확인/취소/메뉴)
10. 세이브: 3슬롯, ~1KB

## 세계관 (반드시 사용)

- **무대**: cafelua — 한글 카페. 카운터, 피아노, 소파, 2층 아틀리에. 사계절 배경.
- **주인공**: luke (카페 주인)
- **조연**: Alpha (메이드 AI, 감정 표현), naia (고양이, 밝기 패턴으로 표현)
- **분위기**: 따뜻한 일상 + 미스터리

## 산출물 요구사항

각 에이전트는 다음을 포함하는 게임 디자인 문서를 생성:

1. **게임 타이틀** (한국어 + 영어)
2. **한 줄 시놉시스**
3. **장르** (텍스트 어드벤처 / RPG / 비주얼노벨 혼합 등)
4. **핵심 메커니즘** (플레이어가 무엇을 하는가)
5. **화면 구성** (각 SceneType에 대한 설명)
6. **챕터 구조** (3-5개 챕터, 각 챕터의 목표)
7. **캐릭터 상호작용** (luke, Alpha, naia 간 관계)
8. **전투 시스템** (Pixel mode에서 어떤 일이 일어나는가)
9. **엔딩 조건** (멀티 엔딩 여부)
10. **플레이 타임 예상** (15-30분 권장)

## 길이 제한

문서당 500-1500자. 과도한 상세보다 실행 가능한 설계 우선.

## 평가 기준

- 80x25 터미널에서 구현 가능한가?
- 한글 자모 타일이 게임플레이에 활용되는가?
- cafelua/naia 세계관이 자연스럽게 녹아있는가?
- 1.44MB 안에 들어오는가?
