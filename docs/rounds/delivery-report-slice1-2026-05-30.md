## 1. 한 줄 요약 (발주자 관점)

> 게임에 들어갈 **한글 자모 5개(ㄱㄴㄷㄹㅁ)의 실제 비트맵**을 만들어 게임에 통합하고 검증했습니다.
> 빌드·테스트·합격기준 모두 통과. **단, 전체 폰트(자모 68 + ASCII 95)의 약 7%일 뿐입니다.**

## 2. 무엇을 납품했나 (계약 대비)

| 계약 항목 | 납품 | 상태 |
|---|---|---|
| 자모 ㄱㄴㄷㄹㅁ 실제 8x16 비트맵 | glyph_data.c 인덱스 95/97/98/100/101 교체 | ✅ |
| brightness 실측 (계약 "on-pixel 실측") | popcount → 16단계 스케일, 전 자모 일관 | ✅ |
| 1.44MB 예산 (C1) | 27,328 bytes = **예산의 1.9%** | ✅ 여유 |
| 독립 실행 (C2) | C99 단일 바이너리 | ✅ |
| 재현 가능 | `tools/gen_jamo.py` 생성기 (손편집 아님) | ✅ |

## 3. 발주자가 직접 검수하는 법 (재현 가능)

```bash
cd projects/alpha-n-naia-144
python3 tools/verify_glyphs.py    # → VERDICT=PASS (합격기준 기계검증)
make && make test                 # → 27328 bytes, 17/17 PASS
python3 tools/gen_jamo.py         # 재생성 (멱등 — 같은 결과)
# 자모 모양 육안 확인: docs/rounds/jamo-bitmap-review-2026-05-30.md 의 렌더
```

검수 포인트: `verify_glyphs.py` 가 PASS 인가, 렌더된 5개가 실제 ㄱㄴㄷㄹㅁ 모양인가.

## 4. ⚠️ 미납·잔여 (정직 — 자기변호 안 함)

| 미납 | 비중 |
|---|---|
| 나머지 자음 14 + 모음 21 + 겹자모 28 = **자모 63개 실제 비트맵** | placeholder(기하패턴) 그대로 |
| **ASCII 95개 글리프** | 전부 blank (미구현) |
| 게임 콘텐츠(시나리오/전투/세계관)·실제 한글 텍스트 렌더 시연 | 미착수 |

**완성도: 전체 폰트의 ~7% (5/68 자모, 0/95 ASCII).** 이건 *파이프라인·도구·게이트를 확립한 슬라이스 1*이지, 납품 가능한 완제품이 아닙니다.

## 5. 검수 의견 요청 (발주자 결정 포인트)

1. 슬라이스 1 품질(자모 5개 모양·검증)이 **인수 기준에 부합**하는가?
2. 부합하면 **같은 파이프라인으로 잔여 자모/ASCII 슬라이스 진행** 승인.
3. ㄱㄹ 등 모양에 수정 의견 있으면 반영 후 재납품.

## 6. 알려진 한계·리스크

- 적대 리뷰(생성 단계)가 cycle#1 에 `font_get_glyph` 를 "미구현"으로 **오판**(font.c 못 봄) — 검증층은 부분 컨텍스트면 틀릴 수 있음. 이번 슬라이스는 풀 컨텍스트 + 기계검증으로 보완.
- 8x16 저해상도라 자모 디테일 한계(ㄹ 등 복잡 자모는 굵기/여백 재조정 여지).
- brightness 스케일(`*15/128`)은 균등 양자화 — 시각 가중 필요 시 재정의 가능.
