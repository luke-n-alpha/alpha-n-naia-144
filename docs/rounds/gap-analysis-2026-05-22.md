## 근본 원인

**하네스 런타임이 없었다.**

프로세스가 문서로만 존재하고 실행을 강제하는 코드가 없었기 때문에:
- 역할 위반을 아무도 막지 못함
- 게이트를 건너뛰어도 시스템이 멈추지 않음
- 회고를 안 해도 다음 단계로 넘어감
- 에이전트가 불참해도 폴백이 안 돌음

문서로 된 규칙은 사람이 지키려고 해도 잊는다.
코드로 된 규칙은 잊을 수 없다.

## 개선안

1. 하네스 런타임(run.py)이 게이트를 강제 — 통과 못 하면 다음 phase 잠금
2. 역할 위반 감지 — OC가 src/ 파일 편집 시 차단
3. 회고 체크 — phase 종료 시 회고 파일 없으면 잠금
4. 폴백 체인 — 에이전트 불참 시 재시도 → 가상 에이전트 → human 개입
5. Receiver Proxy — 인도 전 발주자 관점 사전 검수

## 교훈

- 아는 것과 하는 것은 다르다
- 프로세스는 사람이 지키는 게 아니라 시스템이 강제하는 것이다
- 회고는 나중에 할 게 아니라 지금 해야 한다
- "빨리"가 "제대로"를 이기면 둘 다 잃는다
