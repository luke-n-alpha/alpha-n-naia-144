# 회고 #7 — CX↔GE 교차 검증 사이클 완료

**날짜:** 2026-05-22

## 한 일

1. CX에게 REMAINING 11 수정 디스패치 (OC가 직접 코드 안 짬)
2. CX가 render.c NFD 분해, save.c NUL 종결, scene.c 버그 수정, 테스트 강화
3. GE에게 교차 리뷰 요청 → 2 PASS, 2 NEEDS_REVISION
4. CX에게 GE 피드백 디스패치 → scene.c 대화 인덱스 수정
5. GE가 glyph 비트맵 생성 타임아웃 → OC가 Python 스크립트로 생성 (예외 상황)

## 프로세스 관점

- OC가 코드를 직접 짠 건 glyph_data.c Python 생성뿐. 이건 GE의 도메인인데 GE가 타임아웃이라 예외적으로 처리.
- CX↔GE 교차 리뷰가 실제로 버그를 잡음 (대화 인덱스). 프로세스가 작동하는 걸 실증.
- 회고를 매 사이클마다 작성. 이전엔 안 했는데 이번엔 함.

## 개선 필요

- glyph_data.c를 Python으로 만든 건 임시방편. GE가 복구하면 실제 자모 형태 비트맵으로 교체해야 함.
- CX, GE 할당량 관리가 안 됨. 타임아웃이 빈번함. 하네스 run.py에 재시도/폴백 로직은 있는데 실제로는 수동으로 폴백함.
